SmartMunimJi Backend API Documentation

This document lists all API endpoints for the smartmunimji-backend application, their HTTP methods, responsible controllers, descriptions, request bodies, responses, and required roles.

API Endpoints

Endpoint Method Controller Description Request Body Response Role Required
/sm/register POST CustomerController Registers a new customer in smartmunimji_db.customers. {"name": "string", "email": "string", "password": "string", "phone": "string", "address": "string"} 200 OK: "Customer registered successfully"
400 Bad Request: "Email already exists"		 None
/sm/authenticate POST CustomerController Authenticates a customer and returns a JWT token. {"email": "string", "password": "string"} 200 OK: JWT token
401 Unauthorized: "Invalid credentials"		 None
/sm/cust/profile GET CustomerController Retrieves customer profile (placeholder). None 200 OK: "Customer profile accessed successfully"
401 Unauthorized: "Unauthorized"		 ROLE_CUSTOMER
/sm/update-profile PUT CustomerController Updates customer profile (name, phone, address). {"name": "string", "phone": "string", "address": "string"} 200 OK: "Profile updated successfully"
400 Bad Request: "Customer not found"
401 Unauthorized: "Unauthorized"		 ROLE_CUSTOMER
/sm/admin/register POST AdminController Registers a new admin in smartmunimji_db.admins. {"email": "string", "password": "string"} 200 OK: "Admin registered successfully"
400 Bad Request: "Admin email already exists"		 None
/sm/admin/authenticate POST AdminController Authenticates an admin and returns a JWT token. {"email": "string", "password": "string"} 200 OK: JWT token
401 Unauthorized: "Invalid credentials"		 None
/sm/admin/dashboard GET AdminController Accesses admin dashboard (placeholder). None 200 OK: "Admin dashboard accessed successfully"
401 Unauthorized: "Unauthorized"		 ROLE_ADMIN
/sm/seller/register POST SellerController Registers a new seller in smartmunimji_db.sellers. {"sellerName": "string", "sellersEmail": "string", "password": "string", "sellerContact": "string", "shopName": "string", "shopAddress": "string", "city": "string", "pincode": "string", "category": "string"} 200 OK: "Seller registered successfully"
400 Bad Request: "Seller email already exists"		 None
/sm/seller/authenticate POST SellerController Authenticates a seller and returns a JWT token. {"sellersEmail": "string", "password": "string"} 200 OK: JWT token
401 Unauthorized: "Invalid credentials"		 None
/sm/customer/register-product POST ProductController Verifies order in radhe_shyam_db.orders, checks shop name in smartmunimji_db.sellers, and registers product in smartmunimji_db.registered_products. {"orderId": "string", "dateOfPurchase": "YYYY-MM-DD", "shopName": "string"} 200 OK: "Product registered successfully"
400 Bad Request: "Invalid shop name", "Invalid order ID or date of purchase", "Product already registered"
401 Unauthorized: "Unauthorized"
500 Internal Server Error: "Product not found"		 ROLE_CUSTOMER

Notes

  • Authentication: Protected endpoints require a JWT token in the Authorization header (Bearer <token>).
  • Database:
    • smartmunimji_db: Stores customers, admins, sellers, registered_products.
    • radhe_shyam_db: Stores orders and products (SELECT-only access).
  • Error Handling: APIs return appropriate HTTP status codes and error messages.
  • Logging: All API calls are logged for debugging purposes.
